# Примечания и допущения
- Hazard rate исключён (правка наблюдателя), не рассчитывается и не описывается.
- Источник данных: файлы CSV/JSON, ручной ввод; интеграция с MOEX ISS не заявлена.
- Требование «до 1000 записей ≤5с» снято; производительность зависит от размера сценариев/портфеля.
- Historical VaR/ES считаются на векторе сценарного PnL (demo/simulated mode), а не на реальном историческом time-series; в UI это явно помечается.
- Historical VaR использует дискретную Excel-подобную конвенцию без интерполяции: `k = ceil(N * (1 - CL))`, `VaR = -sorted_pnl[k-1]`.
- При наличии `probability` у сценариев historical VaR/ES считаются как weighted-оценки по левому хвосту `1-CL` (веса нормализуются на сумму).
- Historical ES считается как средний убыток по худшему хвосту (`k` наблюдений), включая VaR-точку.
- Параметрические VaR/ES рассчитываются по одностороннему квантилю Normal с учетом горизонта: `mu_T = mu*T`, `sigma_T = sigma*sqrt(T)`.
- Для параметрического хвоста доступен режим `cornish_fisher` (консервативная поправка к хвостовому квантилю на основе skew/kurtosis).
- LC VaR приведён к денежным единицам базовой валюты: `LC VaR = VaR + Liquidity add-on`.
  Поддерживаются 3 модели add-on:
  1) `fraction_of_position_value`: `haircut * |position_value|`
  2) `half_spread_fraction`: `0.5 * haircut * |position_value|`
  3) `absolute_per_contract`: `|qty| * haircut`
- Валюта должна быть ISO 4217; в расчёте добавлены `base_currency` и `fx_rates`.
  При смешении валют без полного FX-покрытия добавляется предупреждение, а fallback-курс `1.0` помечается как риск методологической ошибки.
- `rate_shift` в стрессах трактуется как абсолютный сдвиг ставки (в долях), `volatility` после шока ограничивается снизу (`max(vol, eps)`).
- Монте‑Карло использует фиксированный seed для воспроизводимости; его можно переопределить на уровне вызова функции.
- Форматирование чисел в отчётах не влияет на вычисления: расчёты ведутся в float64 без округлений.
- UI фронта работает в двух режимах:\
  1) demo (по умолчанию) — всё в браузере, без бэкенда; \
  2) API — через локальный FastAPI (`backend/option_risk/api.py`) + Vite proxy `/api → :8000`.\
  В `run_all.sh` фронт запускается в режиме API (`VITE_DEMO_MODE=0`).
- PostgreSQL/Redis в текущей поставке не используются: хранение снапшотов/кеш — следующий этап (если потребуется).
